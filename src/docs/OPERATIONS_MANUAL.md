## 系统角色说明

| 角色 | 标识 | 说明 |
|------|------|------|
| 超级管理员 | super_admin | 拥有系统全部权限，可管理所有项目、用户和配置 |
| 管理员 | admin | 可管理系统配置、用户角色、项目模板 |
| 项目组长 | pm | 负责单个项目的整体管理、任务分配、进度跟踪 |
| 开发 | developer | 负责具体任务的开发和实现 |
| 测试 | qa | 负责质量验证、测试用例执行 |
| 产品 | product | 负责需求分析、产品规划 |
| 协作方 | collaborator | 外部协作者，权限受限 |

## 超级管理员操作指南

### 系统管理
- 访问「后台管理」模块进行系统配置
- 管理用户账号、角色分配
- 配置系统通知策略
- 审核权限变更申请
- 查看审计日志

### 项目管理
- 创建、编辑、删除项目
- 分配项目负责人
- 查看所有项目的进度和状态
- 导出全局报表

### 入口路径
- 后台管理：`/admin`
- 系统设置：`/settings`
- 全局报表：`/reports`
- 审计日志：`/admin/logs`

## 项目组长操作指南

### 项目看板
- 访问项目看板查看任务分布
- 看板列：待开始 → 进行中 → 待审核 → 已完成
- 点击任务卡片可查看详情或编辑
- 任务状态通过颜色和标签区分

### 任务管理
- 创建新任务：填写标题、描述、优先级、负责人、审核人
- 编辑任务：修改任务的任意属性
- 删除任务：在任务详情页删除
- 提交审核：完成任务后提交给审核人
- 审核任务：审核人可通过或驳回任务

### 成员管理
- 邀请成员：点击「邀请成员」选择组织架构中的用户
- 编辑成员：修改成员的项目角色、工作负载、专注领域
- 移除成员：将成员从项目中移出

### 文档管理
- 上传项目文档
- 分类管理：需求评审、联调验证、测试报告等

### 风险管理
- 识别并记录项目风险
- 评估风险等级：紧急/高/中/低
- 制定风险应对策略

### 入口路径
- 项目看板：`/project/{id}/kanban`
- 项目成员：`/project/{id}/members`
- 项目文档：`/project/{id}/docs`
- 项目风险：`/project/{id}/risk`

## 项目成员操作指南

### 个人工作台
- 查看分配给自己的任务
- 编写并提交日报/周报
- 草稿自动保存：编辑日报时系统自动保存草稿
- 定时发送：可设置日报在指定时间发送
- 发送对象：选择日报的接收人和抄送人

### 任务处理
- 查看任务详情：点击任务卡片查看完整信息
- 更新任务进度：在任务详情页更新进度
- 提交审核：完成任务后点击「提交审核」
- 查看审核结果：收到通知后查看审核状态

### AI 助手功能
- AI 任务拆解：自动分解复杂任务为子任务
- AI 生成自评：根据 PBC 内容自动生成自评
- AI 趋势分析：查看任务完成趋势图

### 通知中心
- 查看系统通知
- 处理待审核任务通知
- 查看任务分配通知
- 标记通知为已读

### 入口路径
- 个人工作台：`/workbench`
- 通知中心：`/notifications`
- 项目看板：`/projects` → 选择项目 → 看板

## 常用操作流程

### 创建项目（管理员/超级管理员）
1. 进入「项目列表」
2. 点击「新建项目」
3. 填写项目名称、描述、起止时间
4. 选择项目成员和负责人
5. 点击「创建」

### 分配任务（项目组长）
1. 进入项目看板
2. 点击「创建任务」按钮
3. 填写任务信息：标题、描述、优先级、负责人、审核人
4. 点击「创建」

### 提交日报（所有成员）
1. 进入「个人工作台」
2. 选择日报模板（日报/周报/项目进度汇报）
3. 填写完成内容、未完成内容、需要协调内容
4. 点击「添加发送对象」选择接收人
5. （可选）设置定时发送时间
6. 点击「发送」或「保存草稿」

### 审核任务（项目组长）
1. 收到任务审核通知
2. 进入通知中心或任务详情页
3. 查看任务内容和完成情况
4. 选择「通过」或「驳回」并填写意见
5. 任务状态自动更新

## 注意事项

1. **权限控制**：不同角色可访问的功能不同，超出权限的操作会被拦截
2. **数据保存**：日报编辑时系统自动保存草稿，刷新页面不会丢失数据
3. **通知时效**：任务审核结果会实时推送到通知中心，请及时处理
4. **定时发送**：定时发送的日报会在设定时间自动发送，请确保网络连接正常
